# 🌳 FamilyBook v4.0.0 (Triple-S Rebuild & Background Engine)

Семейная социальная сеть нового поколения премиум-класса. В версии **v4.0.0** проект перешел на полностью асинхронную архитектуру с распределенной очередью задач, тактильным премиум-визуалом и отказоустойчивой системой Web Push уведомлений. 🚀

---

## ❓ Что нового в v4.0.0?

* **⚙️ Бэкенд-инфраструктура и очереди задач:**
  - Интегрирована распределенная очередь задач на базе **Celery + Redis** для выноса ресурсоемких операций из главного потока приложения.
  - Потоковая изоляция I/O операций реализована через `asyncio.to_thread`, что гарантирует отзывчивость Event Loop FastAPI.
  
* **💾 Оптимизация памяти (No Redis Memory Bloat):**
  - Реализован паттерн общих путей файлов (shared volumes) во временных папках на диске. FastAPI мгновенно сохраняет загружаемые оригиналы и передает Celery только строковые пути. Celery сжимает изображения через Pillow и удаляет временные файлы. Брокер Redis остается чистым от бинарных данных.

* **🎨 Фронтенд и UX-эффекты:**
  - **Иерархические ветки комментариев**: Древовидная структура ответов с рекурсивным обходом в Jinja2 и отступами исключительно на базе Tailwind CSS (`pl-6 border-l border-white/5`). Alpine.js управляет мгновенным открытием мини-форм для ответов.
  - **Честный Ripple-эффект**: Эффект расходящейся золотой волны на кнопках реакций и лайков взрывается строго из координат клика мыши или тапа пальца `(e.clientX, e.clientY)`.
  - **Интерактивный Marble-фон**: Фоновый Marble-градиент плавно смещается вслед за курсором мыши на ПК (через Alpine.js) или совершает мягкий дрейф по CSS-таймеру на мобильных устройствах.

* **📱 PWA & Web Push:**
  - Полноценная поддержка Web Push API в автономном фоновом режиме. Браузерные Push-уведомления доставляются, даже когда телефон заблокирован или вкладка закрыта.
  - Автоматическая очистка невалидных токенов подписки в базе данных при возникновении `WebPushException` (статусы 404 и 410 Gone).

## 🛠 Технологический стек

* **Backend:** FastAPI (Python 3.13), SQLModel, Celery, Redis, Alembic (чистые миграции схемы БД).
* **Frontend:** Alpine.js (интерактивный реактивный визуал), Tailwind CSS (дизайн и иерархические отступы), HTMX (быстрый SPA-эффект).
* **Realtime & Push:** WebSockets, Web Push API (`pywebpush`).
* **DevOps:** Docker Compose (сервисы `web`, `celery_worker`, `redis`, `nginx`), Poetry, Ruff.

## ⚙️ Переменные окружения (.env)

| Ключ | Описание |
| :--- | :--- |
| `DATABASE_URL` | PostgreSQL (Prod) / SQLite (Dev) |
| `SECRET_KEY` | Ключ для подписи JWT-токенов |
| `BOT_TOKEN` | Токен Telegram-бота для алертов Sentinel |
| `ADMIN_CHAT_ID` | Ваш Telegram ID для связи с Sentinel |
| `ENV` | production, development или testing |
| `CELERY_BROKER_URL` | Адрес Redis-брокера (`redis://redis:6379/0`) |
| `CELERY_RESULT_BACKEND` | Адрес бэкенда результатов Celery |
| `VAPID_PUBLIC_KEY` | Публичный VAPID-ключ для Web Push |
| `VAPID_PRIVATE_KEY` | Приватный VAPID-ключ для Web Push |
| `VAPID_CLAIM_EMAIL` | Контактный email для VAPID-подписи |

## 📦 Быстрый старт (Docker Compose)

Запуск всей инфраструктуры (веб-сервер, воркер Celery, база данных, кэш Redis и Nginx прокси) одной командой:

```bash
docker-compose up --build
```

Приложение будет доступно по адресу: http://localhost:8080

## 🛡️ Архитектура безопасности (Sentinel)

Система безопасности работает в три эшелона:
1. **Middleware**: Фильтрация входящих IP-адресов, Anti-XSS пре-валидация и Anti-SQL инъекции.
2. **Honeypot**: Скрытые поля в формах регистрации для мгновенной блокировки ботов.
3. **Global Exception Handler**: Изоляция критических ошибок сервера, автоматическая отправка отчетов администратору в Telegram-бот и плавный редирект пользователя в «безопасную зону».
